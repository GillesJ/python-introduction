Het doel van mijn project is uitzoeken op welke website een fysiek boek het goedkoopste is. De websites die doorgaans het goedkoopste zijn, zijn bol.com, bookdepository en amazon.de. Na heel wat prutswerk bleek echter dat de code van amazon.de zodanig ingewikkeld in elkaar zit, dat ik deze website heb laten vallen. Voor de andere twee websites werkt de code wel. 

In de zipmap zitten in totaal vier bestanden #BD vijf bestanden: deze readme, originele bolcom.py, originele bookdepository.py, alles.py en het uiteindelijke resultaat wheredoibuythecheapestbook.py. In de eerste drie bestanden zit de code waarmee ik begonnen ben. Die van bol.com werkt via het notebook en via het command center (Microsoft Windows), maar die van bookdepository werkt alleen via de notebooks. Alles.py kan opgeroepen worden in het notebook en in het command center, maar het deel van bookdepository werkt niet in het command center. Mijn uiteindelijke code, wheredoibuythecheapestbook.py, kan aangeroepen worden in het notebook en het command center zonder errors (normaal gezien). #BD waarom zijn er verschillende versies? Waarom niet enkel het laatste?

Enkele opmerkingen:
- bij bol.com heb ik alleen het eerste resultaat bij de zoekresultaten opgenomen in de code, omdat dit telkens het goedkoopste (fysieke) boek is [lijn 12]. Bij bookdepository werden alle resultaten opgeroepen, omdat dat daar niet altijd het eerste zoekresultaat het goedkoopste is, maar vaak het tweede of het derde zoekresultaat [lijn 25]. Op het einde van de code [lijn 40 + 41], zie je dat ik alleen de eerste drie resultaten van bookdepository aanroep, omdat de resultaten die daarna komen vaak hardcovers zijn, en dus sowieso duurder en dus niet relevant, ofwel zijn het e-books, die doorgaans veel goedkoper zijn dan de paperbacks waar ik met deze code eigenlijk naar op zoek ben. 
- er wordt in mijn code geen rekening gehouden met verzendingskosten
